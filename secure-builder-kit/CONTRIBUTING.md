# Contributing to secure-builder-kit

Contributions must prioritize defensive education, legal safety, and reproducibility.

## Non-Negotiable Rules

- Authorized-use only content
- Explicit ethics/legal boundaries in every offensive-leaning module
- Safe defaults that target only local labs (DVWA or included demo apps)
- Clear remediation guidance with every demonstrated weakness

## Module Contribution Checklist

- [ ] README includes legal and ethical boundaries
- [ ] Setup steps are reproducible
- [ ] Code examples are functional (no placeholders)
- [ ] Defensive fixes are documented and implemented
- [ ] At least one test is included
- [ ] `architecture.md` contains a Mermaid diagram

## Pull Request Requirements

1. Explain educational objective.
2. List environments tested.
3. Confirm authorized-target restrictions.
4. Include test command/output summary.
